# Generation Methods

TeCoD exposes automatic generation plus explicit method control through `GenerationRequest.method`.

## Method Summary

| Method | Description | Typical Use |
| --- | --- | --- |
| `auto` | Uses retrieval and NLI to choose between template-guided and example-guided generation. | Default production path. |
| `gcd` | Partitioned guided constrained decoding using a compiled template. | Local models with strong template match. |
| `base-gcd` | Whole-sequence grammar-constrained generation from a selected template. | Local constrained baseline. |
| `sgc` | Soft grammar-constrained prompting. | Template guidance with more flexibility. |
| `icl` | In-context learning from retrieved examples. | Novel query structures or API providers. |
| `zs` | Zero-shot SQL generation from schema and query. | Simple baseline or no examples. |

## Auto

`auto` retrieves similar examples, scores them with NLI, and routes high-confidence template matches to constrained decoding. Lower-confidence or structurally novel queries fall back to prompt-based generation with in-context examples.

## GCD

Guided constrained decoding uses compiled templates from `data/c_templs/`. The fixed SQL structure comes from the template, while literal slots are generated by the local model under regex constraints. This path requires direct model logits, so it is only available for local providers.

## Base-GCD

Base-GCD converts a selected template into a grammar constraint and applies it over the whole SQL generation. It is useful when you want grammar-constrained output without partitioned decoding.

## SGC

Soft grammar constraining uses template information as prompt guidance rather than a hard decoding constraint. It can be used when a template is helpful but the target SQL may need to differ from it.

## ICL

In-context learning prompts the model with retrieved question-SQL examples. It is the main flexible path for OpenAI-compatible providers.

## ZS

Zero-shot generation uses the schema and natural-language query without retrieved examples. It is most useful as a simple baseline or for low-latency experiments.

## Explicit Method Example

```python
from src.api import TeCoD
from src.models.data import GenerationRequest

with TeCoD(
    config_overrides=["+data=financial"],
) as tecod:
    request = GenerationRequest(
        query="How many accounts from north Bohemia are eligible to receive loans?",
        method="icl",
    )
    result = tecod.generate_with_method(request)
    print(result.pred_sql)
```
